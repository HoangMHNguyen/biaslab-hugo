Teardown API

Any subscription-like object should implement a valid teardown logic.

Example

using Rx

struct MuCustomSubscription <: Teardown
    # some fields here
end

Rx.as_teardown(::Type{<:MuCustomSubscription}) = UnsubscribableTeardownLogic()

function on_unsubscribe!(subscription::MyCustomSubscription)
    # dispose resources here
end

Rx.as_teardown â Function
as_teardown(::Type)

This function checks teardown trait behavior specification. Should be used explicitly to specify teardown logic trait behavior for any object.

Examples

using Rx

struct MySubscription <: Teardown end

Rx.as_teardown(::Type{<:MySubscription}) = UnsubscribableTeardownLogic()
Rx.on_unsubscribe!(s::MySubscription)    = println("Unsubscribed!")

subscription = MySubscription()
unsubscribe!(subscription)
;

# output

Unsubscribed!

See also: Teardown, TeardownLogic

Rx.UnsubscribableTeardownLogic â Type

Unsubscribable teardown logic trait behavior. Unsubscribable teardown object must define its own method for on_unsubscribe!() function which will be invoked when actor decides to unsubscribe! from Observable.

See also: TeardownLogic, on_unsubscribe!, unsubscribe!

Rx.on_unsubscribe! â Function
on_unsubscribe!(teardown)

Each valid teardown object with UnsubscribableTeardownLogic trait behavior must implement its own method for on_unsubscribe!() function which will be invoked when actor decides to unsubscribe! from Observable.

See also: Teardown, TeardownLogic, UnsubscribableTeardownLogic

Void teardown logic trait behavior. Void teardown object does nothing in unsubscribe! and may not define any additional methods.
